背景

随着各种MMD工具的更新，插件的出现，制作MMD有了更多选择。
比如从传统的MikuMikuDance到Blender、Unity和Unreal Engine等专业3D软件。

难点和问题

最大的问题其实是数据格式，在MMD软件中，模型文件是pmx格式，而动作和相机数据都是vmd。

PMX作为MMD模型的专有格式，虽然在MMD生态系统中被广泛支持，但在其他3D软件中却面临诸多问题：

  • ​材质系统差异​：PMX的材质表达与主流PBR材质工作流不兼容
  • ​骨骼权重限制​：某些复杂蒙皮效果在其他引擎中难以完美重现
  • ​物理系统​：MMD特有的物理模拟在其他平台需要重新实现

VMD格式虽然小巧高效，但也存在跨平台问题：

  • ​时间轴精度​：不同软件对帧率的处理方式不同
  • ​曲线插值​：动作曲线在不同引擎中的表现可能不一致
  • ​相机动画​：MMD相机参数与其他3D软件的映射问题

解决方案

  1. ​PMX转通用格式​：
    • 使用Blender的MMD Tools插件转换为fbx/gltf
    • CATS插件提供的优化和重定向功能
  2. ​VMD动作处理​：
    • MMD4Mecanim等Unity插件提供的重定向解决方案
    • 在Blender中烘焙到一个物体
  3. ​VMD相机运动处理​：
    • 使用MMD Tools导入VMD相机数据，会生成一个带有关键帧的空对象和MMD相机，将它烘焙并到处

新MMD流程

Blender

  • 在Blender上我们可以用鸣潮二创也能如此简单🤨？！！【鸣潮二创教程/偏新手】哔哩哔哩_bilibili这篇教程提供的整合包。（由于鸣潮二创也基本上就是MMD制作的流程，所以这篇教程大概够了。

我再精简一下这个视频的内容：

  • 下载好这个定制版blender之后，导入启用三个插件
    • mmd-tool
    • Miku_Miku_Rig2.0 (AAAMi)
    • mmr补充4.1 (鬼离果)
  • 然后进行简单的设置（为了适应预设效果）
    • 使用合成器
    • 打开后处理
    • 修改色彩方案
    • 视图降噪关了
  • 然后追加内容，先导入NodeTree里的Bloom，再导入Collections的模型
    • 用mmd-tool处理
    • 合成器里面加入Bloom节点
    • 用MMR生成控制器
    • 用MMR补充插件删除多余部分
  • 然后去时间轴用i插入关键帧k动画或者mmd-tool导入动画
  • mmd-tool生成物理以及烘焙物理
  • 可以调整内容
    • 修改边缘光光照颜色和强度
    • 显示修改器效果
    • 帧率和关键帧距离都放大

mmd-tools使用注意

  • 导入动作可以设定间距，但是必须当前帧是1。而比如间隔10帧，则第11帧的动作开始有效。

换头手术

  • 首先用MMD-tools将模型的头骨切开，删除不要的部分，注意刚体啥的也要一块删

  • 保留需要换的头的一部分脖子用于过渡

  • 将头和脖子合并，并且定点也要合并（按距离）

  • 然后解锁骨骼的移动权限，将头调整到合适位置

  • 然后姿态模式下将两个骨头缝合

  • 测试一下如果没有联动，就是父子级设置错了，要后选父级，可以去编辑模式改

  • 换头后端动画问题：

    • 如果看不到关键帧，就去动画时间线把报错取消掉
    • blender种骨骼姿态也可以应用为形态键
    • 所以有时候形态键冲突导致新头没有表情，可以通过mmdtools里的变形工具选项将形态键应用为.placeholder，这样就有了完整的形态键，可以去看动作文件把需要用于表情的锁定，然后删除其他的。一般，头上的会自动重命名加个2，（其实合并之前应该先一键删除无用形态键）

增加细节

  • 按距离合并定点
  • 重置矢量 (Alt + M 或者 编辑模式去网格里面找矢量选项）
  • 去面里面找 三角面->四边面 （选择90°，比较UV）
  • 再加细分修改器，矫正平滑修改器

贴花

给人物装上配饰有几种方法：

  • 先选择身体再选择配饰，进入刷权重界面，进行权重传递，合并网格前需要保证UVMap名字一样（网格需要在骨骼下面？）
  • 通过表面形变修改器吸取物体，点击绑定但是不要应用

布料模拟

布料模拟，建议素体以角色为基础，然后导出素体abc时只选择你要模拟的部位，因为摇晃等不确定因素可能导致素体和角色动作不相同从而造成衣服严重穿模，同时只有一个角色也便于添加物理

删掉手臂烘焙动骨动画，导出仅选择，为abc

导入MD的时候单位mm缩放是125000%

解算完选中服装板片导出pc2

导入blender obj文件缩放0.0008

添加网格缓存修改器并添加数据

打光和输出配置

可以创建多个立方体做体积雾（原理化体积）
需要把世界环境的雾生成范围改一下
为了速度快一点，可以把渲染的采样数量降低，光阈值提高
可以把效果设置为电影感，同时改为高对比度
伽马值适当调低

渲染之前检查一下表情有没有应用

Unity

依赖Blender

先在Blender导出FBX格式的数据：

  1. 选择要导出的模型和骨骼系统
  2. 点击”文件”->”导出”->”FBX”
  3. 在导出设置中配置以下关键选项：
    • 勾选”仅导出选中对象”
    • 包含物体类型选择”Mesh”和”Armature”
    • 几何数据中确保包含法线和UV
    • 如果包含动画，勾选”烘焙动画”
  4. 将FBX文件和贴图拖入Unity项目Assets文件夹
  5. 选中FBX文件，在Inspector中进行以下设置：
    • 动画类型设置为”Humanoid”（人形骨骼）
    • 配置骨骼映射关系
    • 设置材质和贴图

PS： 以上操作其实一般不用于Unity，因为Unity的动作模型动作导入用MMD4UnityTools插件更加便捷和准确，Blender导出镜头部分则主要在Unreal那边统一讲解。

不依赖Blender

主要使用MMD4UnityTools工具集，安装插件后将需要转化的文件拖入到资源文件夹即可进行进一步操作。
我之前也编写过一个 自动取出主贴图来适应Unity新渲染管线的插件

Unreal

依赖Blender

同样有使用Blender和不使用Blender的方法，使用的话和Unity类似，但是要注意单位和导出的问题

人物模型部分

  • Blender
    • 安装MMD Tools插件
    • 把帧率改一下，缩放单位改为0.01
    • MMD Tools导入PMX并把缩放改为8 （如果要保持和VRM4U导入骨骼一致用兼容骨架，可以取消勾选将骨骼重命名，兼容骨架需要手动在虚幻骨架窗口中 window -> asset detail 来打开资产详情）
    • 不选中rigidbody和joints层级，仅导出可见项，记得骨骼可见（VRM4U导出的不包含rigidbody，从mesh开始的，所以可以只选中这一层级）
    • 按照UE骨骼标准调整骨骼父级关系
    • 同时选中模型和骨骼导出为FBX
    • 路径模式选择”复制”并内嵌纹理（如果还是没有贴图，那就取消自动打包资源再试试。。如果还不行，可能是shader节点不支持FBX，比如没有用BSDF，可以选中mesh然后点击convert to blender）
    • 设置平滑选择为面
    • 取消勾选”添加叶骨”
  • UE
    • 提前创建masked类型材质并用变量赋予有rbga的black贴图，rgb a分别连到颜色和透明度及mask
    • 导入时勾选高级里的Vertex Attributes
    • 选择法线和切线
    • 选择实例化材质，并且赋予base texture给基础和不透明度

动作部分

  • 需要接着人物模型部分操作
  • 选中角色骨骼导入动作，缩放改为8
  • 将动作赋予给人物后，可以启用物理和烘焙，导出的时候只选择骨骼和烘焙动画，如果需要表情，需要导出网格体

相机部分

  • Blender
    • 单位改为0.01，选中相机，导入相机动作缩放为8
    • 导出.usdc （通用场景描述）
    • 勾选动画以及注意使用英文路径和名称
  • UE
    • 启用USD导入插件
    • 角色设置到原点
    • file -> import into level 导入 usdc文件
    • 找到资源视图里的关卡序列，并添加Camera Cut Track 选择为 Camera
    • 然后把角色和动作导入进去
    • 播放

VRM4U插件

启用插件后要在项目设置里面拉高他们的导入优先权

使用VRM4U插件导入PMX：

  • 材质类型选择MToon Unlit
  • scale 0.08
  • 生成描边
  • 勾选部分Optimize选项
  • 如果要启用描边，导入后把 Plugins/VRM4U Content/Util/Actor/MToonAttachActor 拖到人物位置，并且在细节面板选择人物。要改参数可以去人物AssetList里面找Outline材质

使用IVP5U导入VMD：
有可能出现重定向问题，可以重新设置重定向animation，不建议

启用物理

主要分为从Blender，Unity，UE三个方面

  • Blender中用mmd-tool，注意要在人物移动到开始跳舞的位置再点击物理，然后再点击烘焙，通知注意播放模式是播放每一帧（这是因为Blender的物理计算是依靠刚体，从第一帧开始顺序计算的，不是实时模拟）

  • 可以用spring bones插件去模拟欧派运动，通过选择姿态模式骨骼应用之后开始模拟，然后再烘焙动作。

  • 以下内容为Blender物理指南，作者MiaoBox
    1.开启物理
    2.开启物理后有两个参数，一般用默认的就行
    3.游标拖到0帧（一般第1帧也行）
    4.点下更新世界，改子步数：1，迭代：60
    5.设置下结束帧，设置多长物理就烘培到哪，如果不小心烘焙太长，按esc取消
    6.看下是不是在“播放每帧”，烘培需要每帧
    7.烘培，完成

    雷区1：没有烘培好就去拖动关键帧
    雷区2：动作幅度太大
    雷区3：烘培的起始帧不是0或1

    建议1：
    遇到模型物理一直错误，可能模型的问题，换模型试
    建议2：
    第一帧静止姿态，这也是为什么导入的动作第一帧是静止的，隔几帧之后才是动作开始

  • Unity中就是用Magica插件啥的，具体有点忘了

  • UE用Kawaii插件

输出结果

启用插件movie render queue就可以在sequencer上找到渲染视频的按钮，可以增加几个设置选项，比如输出改为PNG。
导出后的图片序列可以用AE进行处理。

解惑

为什么要改变缩放单位，导入时还要进行缩放？

  • 改变单位，是为了方便导出到其他软件
  • 因为软件只认数值而不认单位
  • Blender使用的是m，而UE使用的是cm
  • 所以缩放改为0.01即可
  • 导入模型要放大100倍是为了适应现在缩放后的单位
  • 否则角色导入进去就太小了

为什么要将路径改为复制并内嵌材质，材质转为Blender？

  • 路径改为复制是为了让材质贴图在其他机器其他路径也能显示
  • 内嵌材质是因为fbx格式模型可以内嵌材质（如果是obj格式就得关闭自动打包资源，改为手动解包，因为blender格式可以内嵌资源，而obj格式不行）
  • 材质转为blender的BSDF才能适配fbx导出时的材质数据，否则一些奇奇怪怪只有blender有的高级功能节点，无法自动转化为fbx的材质，就会导致丢失材质
