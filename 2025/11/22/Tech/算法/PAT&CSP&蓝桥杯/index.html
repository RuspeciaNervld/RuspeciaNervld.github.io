语法

  • 使用reverse函数传入的是迭代器的头尾，需要include <algorithm>
  • 使用istringstream要引入 <sstream>
  • 可以使用to_string方法来把数字转为字符串
  • 不能直接printf xxx.size()，需要强制转为int
  • vector使用的是push_back
  • map插入要么是pair，要么直接用mp[key] = value
  • 给一个没初始化的vector赋值使用 = vector<int>(num,val)
  • 可以用({a,b})这种来构建结构体，或者作为隐式构造函数构造vector
  • vector<高级类型> 高级类型必须有默认构造函数
  • max_element会返回第一个最大值的索引（如果有相同值）

技巧

  • 空间换时间，可以开一个很大的数组，不用担心其中有些没有用到
  • 比较函数可以加一个flag来根据外界条件，做不同的比较逻辑
  • 模拟行为比模拟时间更好，把每个对象对外最重要的信息作为维护点即可
  • 通过记录一个点的所有前驱，可以记录所有最短路
  • dfs可以记录路径然后在最后到达终点的时候处理这个路径的整个信息
    • 或者记录一个信息，不断更新
  • 二叉树递归的时候是一层一层来的，所以只需要在参数那里加个level就可以知道自己是第几层，改为2 x n + 1 和 2 x n + 2就变成了index 了
  • 两次DFS居然就能找到一个图的直径：第一次找到的是整个图直径的一个端点，第二次就能找到整个图的直径的另一个端点
    • 可以理解为先找到一个点，找到离他最远的所有点
    • 然后从离他最远的点里面找一个再去找，可能找到和最开始选的那个点近，但是离其他点都远的点
    • 1021 Deepest Root - PAT (Advanced Level) Practice (pintia.cn)

常见错误

  • 判断浮点型是否相等需要借助 abs(差) < 1e-5
  • 二分查找也可以考虑溢出的情况，另外注意我的统一写法是左闭右开
  • vector的全局变量要用assign再初始化一下，必须两个参数，数量和初值
  • 一些常见类型错误的原因：
    • 段错误：数组越界，没有初始化
    • 运行时错误：没有初始化
  • 注意没有环路的图遍历是不需要还原vis状态的，可以想象成二叉树
  • 一定注意图的索引是不是从0开始
  • 内存超限：可以用一维代替二维
  • 时间超限：可以用数组代替vector
  • 奇奇怪怪？看看读入有没有问题
  • cin不能获取空格，要用getline(cin,line);
    • 注意用getchar() 把上一行的\n读掉

蓝桥杯

赛制OI，中途没法看到提交的结果
策略：先暴力，有时间再根据数据范围来判断时间复杂度

10^5 要优化到nlogn
10^9 即使O(N)也会超时，所以o(1) 或者 o(根号)可能是数学题，可以打表找规律

重点是

  • dfs, 状态压缩dp, dp, 二分, 各种sort, heap,　dijkstra，spfa，BFS，并查集，Kruskal

顺序：基础算法(前缀(异或)和，二分，差分) > DP > 搜索 > 图论(最短路，最小生成树) > 数据结构

做题思路：先想一下暴力解法，然后看看能不能优化，如果不能想到优化，那就先写暴力

暴力的方法：

  • for
  • DFS,BFS
  • 打表找规律

蓝桥杯中常见坑：

  • 必须写return 0
  • 一定要开long long 可以用 typedef long long ll;

  • 需要优化iostream的速度或者用c标准库的
    • ios::sync_with_stdio(0);
    • cin.tie(0); cout.tie(0);
    • 用\n而不是endl

  • 万能头#include<bits/stdc++.h>
  • INF 的值 0x3f3f3f3f

树上差分

  • 边权差分（用每个点来表示他与父亲相连的边的权）
    • 给起点和终点+1
    • 给两个点的最近公共祖先-2
    • 所有差分都完成之后，让每个点都加上子树和
    • 点权就是我们要拿到的边权结果
  • 点权差分

单调队列和滑动窗口结合就可以维护一个窗口中的最值
如果是二维的，先求每一行的结果（窗口大小为子矩阵长），再求每一列的结果（窗口大小是子矩阵高）
