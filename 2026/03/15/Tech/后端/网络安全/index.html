OAuth2.0

  • 你在电商 APP 点 “微信登录” → 电商 APP（客户端）携带自身 appid、回调地址等参数，跳转到微信开放平台（授权服务器）；
  • 你扫码并点击 “允许授权” → 你（资源所有者）完成授权，微信服务器先完成「认证」（确认你是合法微信用户），再完成「授权」（确认你同意电商 APP 获取信息）；
  • 微信开放平台生成临时 Code + 关联你的 openid → 把临时 Code 通过回调地址返回给电商 APP（此时电商 APP 还拿不到 openid，仅拿到 Code）；
  • 电商 APP 后端用 Code + 自身 appid/appsecret，向微信开放平台申请换取 access_token → 微信服务器验证 Code 合法性后，返回核心凭证：
    • access_token（访问令牌，代表电商 APP 有访问你微信信息的权限）；
    • openid（你在该电商 APP 对应的唯一微信标识，微信对用户的匿名身份 ID，永久绑定除非你解绑）；
      （可选返回 refresh_token，用于 access_token 过期后刷新）；
  • **电商 APP 用 access_token + openid**，向微信资源服务器请求你的昵称 / 头像 → 微信服务器校验 access_token 和 openid 匹配性，确认无误后返回你的基础信息；
  • 电商 APP 完成登录：
    • 用 openid 匹配自身数据库（新用户则创建账号，老用户则关联已有账号）；
    • 生成电商 APP 自身的登录 Token（比如 JWT）返回给前端；
      全程没碰过你的微信密码，且 openid 是你在该电商 APP 的唯一身份标识（不同 APP 的 openid 不同，保护你的微信账号安全）。

JWT

JWT是认证，但是实际上里面可以存储一些跟鉴权相关的信息，这样也可以用来鉴权了

1）私钥签名（生成 JWT）

输入：

  1. Header（算法 + 类型）
  2. Payload（用户数据：userId、用户名、过期时间等）
  3. 私钥（Private Key）

做什么：

把前两部分用 Base64 编码后，
用 私钥 + 算法（RS256） 生成一段签名。

输出：

完整 JWT Token

公钥验签（验证 JWT）

输入：

  1. 完整的 JWT Token
  2. 公钥（Public Key）

做什么：

  1. 拆分出：header、payload、signature
  2. 用公钥和算法验证签名是否合法

输出：

  • true → 合法、未篡改
  • false → 伪造 / 被改 / 过期
