首先这个项目的亮点：

亮点

  • 意图识别（多级意图分类与置信度阈值判断）
  • 问题重写（将用户口语化表达改写为适合检索的标准查询），可能类似多agent？
  • 多道并行检索（策略模式）
  • 后处理流水线（去重，重排序）
  • 流式输出
  • 记忆模块（滑动窗口+自动摘要）
  • 缓存
  • 文档入库流水线（获取、解析(Tika)、增强、分块、提取、向量化等节点的灵活配置与条件执行。实现入库任务的精细化管理与异常定位。）
  • 三级熔断机制（CLOSED/OPEN/HALF_OPEN状态切换，故障模型自动降级）
  • 异步线程池矩阵（区分流式输出、意图识别、文档检索等8类任务）这些任务需要开新线程
  • MCP (构建 MCP工具服务化架构，定义MCPToolExecutor 接口规范，支持销售数据查询等多种外部工具扩展。)

核心思考

  • 意图识别
    • 系统包含哪些功能（意图）
  • 问题重写
    • 是多agent实现？（不是，提示词包含删除什么、改成什么风格、格式之类的，本质上是让高级的ai来做简单的自然语言处理任务，因此一些方便ai理解的自然语言任务可以在这一步直接一步到位完成，比如表述规范、指代替换等。）
    • 怎么不浪费token？
  • 多道并行检索
    • 为什么要用策略模式（其实就是借口一致，但是根据不同类型选择具体实现类）
  • 后处理流水线（去重，重排序）
    • 这个有没有算法和指标 采用外部AI模型服务根据query对chunk进行相关性排序（如阿里云百炼），非自研算法

意图识别

  1. 知识库类意图 (KB)
    • 编码：0
    • 用途：走 RAG 检索流程
    • 场景：用户提问需要从知识库中检索相关文档内容
  2. 系统交互类意图 (SYSTEM)
    • 编码：1
    • 用途：处理系统级交互
    • 场景：如欢迎语、自我介绍等系统预设响应
  3. MCP 实时数据交互意图 (MCP)
    • 编码：2
    • 用途：实时数据交互
    • 场景：需要通过 MCP (Model Context Protocol) 获取实时数据

意图树注入prompt提示词，实现可扩展意图识别。构造给 LLM 的 Prompt：

  • 列出所有【叶子节点】的 id / 路径 / 描述 / 示例问题
  • 要求 LLM 只在这些 id 中选择，输出 JSON 数组：[{“id”: “…”, “score”: 0.9, “reason”: “…”}]
  • 特别强调：如果问题里只提到 “OA系统”，不要选 “保险系统” 的分类
  • 如果存在 MCP 类型节点，使用增强版 Prompt 并添加 type/toolId 标识
